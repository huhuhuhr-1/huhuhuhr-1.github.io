核心概念

1. 交互式会话模式

默认模式，适合复杂的开发任务：

特点：

  • 持续的对话上下文
  • 多轮交互
  • 工作区权限管理
  • 会话可恢复

2. 非交互模式（Print Mode）

适合脚本化和批量处理：

特点：

  • 一次性执行
  • 可输出 JSON 格式
  • 适合集成到 CI/CD
  • 跳过工作区信任对话框（仅在可信目录使用）

3. 会话持久化

4. 工作区与权限

Claude Code 使用工作区概念管理文件访问权限：

  • 首次访问目录：会弹出信任对话框
  • 权限模式：可通过 --permission-mode 配置
  • 工具白名单：使用 --allowed-tools 限制可用工具

选项参考

输出格式选项

控制输出格式和行为的选项。

-p, --print

非交互模式，打印响应后退出。

注意事项：

  • 跳过工作区信任对话框，仅在可信目录使用
  • 适合脚本化和自动化

--output-format <format>

指定输出格式（仅配合 --print 使用）。

可选值：

  • text - 纯文本（默认）
  • json - JSON 格式（单个结果）
  • stream-json - 实时流式 JSON

--json-schema <schema>

JSON Schema 验证，确保输出符合指定结构。

--include-partial-messages

包含部分消息块（仅配合 --print 和 --output-format=stream-json）。

--input-format <format>

输入格式（仅配合 --print 使用）。

可选值：

  • text - 纯文本（默认）
  • stream-json - 实时流式 JSON 输入

--replay-user-messages

重新发送来自 stdin 的用户消息（配合流式输入输出）。

调试选项

用于调试和问题排查。

-d, --debug [filter]

启用调试模式，支持类别过滤。

常见调试类别：

  • api - API 调用
  • hooks - 钩子执行
  • statsig - 统计分析
  • file - 文件操作
  • network - 网络请求

--verbose

覆盖详细模式设置。

--mcp-debug

已弃用，请使用 --debug 代替。

会话管理选项

控制和恢复会话的选项。

-c, --continue

继续最近的会话。

-r, --resume [value]

恢复指定会话，或打开交互式选择器。

--fork-session

创建新会话 ID 而不是重用原会话（配合 --resume 或 --continue）。

--no-session-persistence

禁用会话持久化（仅配合 --print）。

--session-id <uuid>

使用指定的会话 ID。

工具权限选项

控制工具访问权限的选项。

--allowedTools, --allowed-tools <tools...>

允许的工具列表（逗号或空格分隔）。

--tools <tools...>

指定可用工具集（仅 --print 模式）。

--disallowedTools, --disallowed-tools <tools...>

拒绝的工具列表（逗号或空格分隔）。

--dangerously-skip-permissions

绕过所有权限检查（仅推荐用于无网络访问的沙箱环境）。

--allow-dangerously-skip-permissions

启用跳过权限检查的选项，但默认不启用。

模型配置选项

控制 AI 模型行为的选项。

--model <model>

指定会话使用的模型。

常用模型别名：

  • sonnet - 平衡性能和速度
  • opus - 最高质量
  • haiku - 最快响应

--fallback-model <model>

启用默认模型过载时的自动降级（仅 --print 模式）。

--agent <agent>

指定会话使用的代理（覆盖 ‘agent’ 设置）。

--agents <json>

定义自定义代理的 JSON 对象。

MCP 配置选项

MCP（Model Context Protocol）相关配置。

--mcp-config <configs...>

从 JSON 文件或字符串加载 MCP 服务器（空格分隔）。

配置文件格式示例 (server.json)：

--strict-mcp-config

仅使用 --mcp-config 指定的 MCP 服务器，忽略其他配置。

系统提示词选项

控制系统提示词的选项。

--system-prompt <prompt>

使用自定义系统提示词。

--append-system-prompt <prompt>

追加系统提示词到默认提示词。

权限模式选项

控制权限检查行为。

--permission-mode <mode>

指定会话的权限模式。

可选值：

  • default - 默认权限模式
  • acceptEdits - 自动接受编辑
  • bypassPermissions - 绕过权限检查
  • delegate - 委托权限决策
  • dontAsk - 不询问权限
  • plan - 计划模式

设置和配置选项

--settings <file-or-json>

从文件或 JSON 字符串加载额外设置。

--setting-sources <sources>

指定要加载的设置源（逗号分隔）。

可选值：

  • user - 用户设置
  • project - 项目设置
  • local - 本地设置

--plugin-dir <paths...>

仅为此会话从指定目录加载插件（可重复）。

Beta 和测试选项

--betas <betas...>

在 API 请求中包含 Beta 标头（仅 API 密钥用户）。

IDE 集成选项

--ide

启动时自动连接到 IDE（如果有恰好一个有效 IDE）。

--no-chrome

禁用 Chrome 集成。

--chrome

启用 Chrome 集成。

目录和会话选项

--add-dir <directories...>

允许工具访问的额外目录（可重复）。

其他选项

--max-budget-usd <amount>

API 调用的最大美元金额（仅 --print 模式）。

--disable-slash-commands

禁用所有斜杠命令。

-v, --version

输出版本号。

-h, --help

显示帮助信息。
